거래선별 원장 매입/매출 필터 근본 수정

매입만 켰는데 매출 행이 남는 문제를 서버 쿼리와 클라이언트 렌더링 양쪽에서 수정했습니다.

대상: 거래선별 원장 DOM 테스트: 통과 서버 필터 테스트: 통과 Build: 통과

최종 원인

기존 서버 필터가 “매입 행”을 purchase_amount != 0으로 판단했습니다. 이관 매출 행에도 원가/매입금액 성격의 값이 들어갈 수 있어, 매입 원장에 매출 행이 섞일 수 있었습니다. 따라서 금액 컬럼 기준이 아니라 direction='purchase' / direction='sale' 기준으로 필터해야 했습니다.

수정 내용

완료서버 쿼리 필터 변경

매입 전용은 direction='purchase', 매출 전용은 direction='sale' 조건을 추가했습니다.

완료잔액 계산용 legacy query 보정

미수/미지급 러닝 밸런스에서도 반대 방향 legacy row가 섞이지 않도록 direction 조건을 추가했습니다.

완료클라이언트 즉시 필터 유지

체크박스 클릭 직후에도 서버 재조회 전 기존 row가 남지 않도록 visibleRows 렌더링을 유지했습니다.

완료실제 DOM 테스트 추가

정적 문자열 검사가 아니라 React 컴포넌트를 jsdom에 렌더링해 매입/매출 row가 실제 DOM에서 숨겨지는지 검증했습니다.
