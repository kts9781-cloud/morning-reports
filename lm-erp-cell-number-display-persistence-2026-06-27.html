CELL 번호 표시·저장 흐름 수정

제품 수불장 CELL 컬럼에 마이너스재고 같은 상태/위치 라벨이 표시되던 문제를 수정하고, 매입전표의 CELL 입력값이 실제 재고 LOT의 stock_lots.cell_no로 저장되도록 보강했습니다.

확인된 문제

  • DB에는 이미 stock_lots.cell_no가 있었지만, 일부 화면/흐름이 과거 필드인 location 또는 stock_location과 섞여 있었습니다.
  • 수불장 CELL 컬럼이 cell_no ?? location 형태로 표시되어, 음수 재고 LOT의 location='마이너스재고'가 CELL처럼 보였습니다.
  • 매입전표 입력칸은 실질적으로 CELL 번호였지만 UI 라벨은 “위치”였고, RPC 생성 직후 LOT의 cell_no 동기화가 누락되어 있었습니다.

수정 내용

영역수정
제품 수불장CELL은 stock_lots.cell_no만 표시합니다. location fallback을 제거했습니다.
매입전표 입력라인 입력 표의 “위치” 헤더를 CELL로 변경했습니다.
매입전표 저장/수정저장/수정 후 purchase_slip_lines.stock_location 값을 연결된 stock_lots.cell_no로 동기화합니다.
legacy location 정리LOT의 location이 동일한 CELL 값이면 null로 비워 중복 저장을 줄입니다.
매입 LOT 연결purchase_slip_lines.created_stock_lot_id를 사용해 현재 매입 행의 LOT/CELL을 정확히 찾습니다.

기존 데이터 처리

기존 매입전표 라인 중 stock_location에는 CELL 값이 있지만, 연결된 stock_lots.cell_no에는 아직 반영되지 않은 데이터가 확인되었습니다.

이 보고서 작업에서는 안전을 위해 기존 DB 일괄 백필은 실행하지 않았습니다. 앞으로 새로 저장/수정되는 매입전표는 자동 동기화됩니다. 기존 데이터까지 즉시 보이게 하려면 별도 백필을 승인 후 실행하면 됩니다.

결론

이제 CELL 컬럼은 실제 CELL 번호만 표시하며, 매입전표에서 CELL 번호를 입력하면 해당 재고 LOT의 cell_no로 저장되는 흐름이 준비되었습니다.
